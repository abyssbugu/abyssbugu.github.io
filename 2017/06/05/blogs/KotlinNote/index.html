2017年 Google I/O 大会,将Kotlin指定为android官方开发语言.整个android开发阵营刮起了Kolin的狂风.本人亦是万分欣喜,终于有一门新语言,可以将我从笨重的Java中解放出来了.拥有java基础学习Kolin并不难,官方文档还有各种学习资料已经写的非常详尽了.本篇文章将记录我学习Kolin以及重构项目的总结的一些Kolin需要注意的点

Kotlin 的空安全

Kotlin有两种类型：一个是非空引用类型，一个是可空引用类型。
对于可空引用，如果希望调用它的成员变量或者成员函数，直接调用会出现编译错误，有三种方法可以调用：

  • 在调用前，需要先检查，因为可能为null
  • 使用b?.length的形式调用，如果b为null，返回null，否则返回b.length
  • 使用b!!.length()的形式调用，如果b为null，抛出空指针异常，否则返回b.length

let函数

首先let()的定义是这样的，默认当前这个对象作为闭包的it参数，返回值是 ==函数里面最后一行==，或者指定return

fun <T, R> T.let(f: (T) -> R): R = f(this)

apply函数

apply函数是这样的，调用某对象的apply函数，在函数范围内，可以任意调用该对象的任意方法，并 ==返回该对象==

with函数

with函数是一个单独的函数，并不是Kotlin中的extension，所以调用方式有点不一样，==返回是最后一行==，然后可以直接调用对象的方法，感觉像是let和apply的结合。

fun <T, R> with(receiver: T, f: T.() -> R): R = receiver.f()

run函数

run函数和apply函数很像，只不过run函数是使用 ==最后一行的返回==，apply返回当前自己的对象。

fun <T, R> T.run(f: T.() -> R): R = f()
